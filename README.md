# DynimicTime simulation

В большинстве симуляций используется дискретизация времени, где состояние объектов изменяется через равные промежутки времени (тики). Эти тики эквивалентны единицам виртуального времени. Например, в симуляции движения частиц каждую единицу времени частица перемещается на расстояние, соответствующее её скорости. Однако такой подход имеет ряд недостатков:

- **Ограничение точности:** При высоких скоростях объекты могут "проскакивать" через препятствия, так как промежуточные положения не обрабатываются.
- **Фиксированный шаг времени:** Все частицы обновляются одновременно, что не всегда отражает физически корректное поведение.

## Предлагаемое решение: динамическое время

Вместо того чтобы фиксировать временные интервалы, фиксируется расстояние, которое проходит объект за тик. Таким образом:

- **Время, проходящее за тик, становится динамичным:** Оно зависит от скорости частицы. Чем выше скорость, тем меньше времени проходит за тик.
- **Частицы с высокой скоростью обрабатываются чаще:** Это позволяет повысить точность симуляции и избежать проблем с "проскоками" через препятствия.

### Ключевая идея: таймлайн событий

1. **Обычные симуляции:** Ключи (обновление одной точки, расчёт её поведения на определённое время) для всех объектов располагаются на таймлайне равномерно. Все объекты обновляются одновременно на каждом шаге, через равные временные промежутки.
![Таймлайн обычных симуляций](https://i.imgur.com/TKv9w7z.png)
2. **Динамическое время:** Ключи распределены по таймлайну неравномерно. Каждый ключ соответствует обновлению конкретного объекта в момент, когда он проходит фиксированное расстояние с момента последнего обновления. Симуляция перепрыгивает от одного ключа к следующему, а не идёт по таймлайну равномерно.
![Таймлайн этой симуляции](https://i.imgur.com/cJnvTy9.png)

### Преимущества подхода

1. **Повышенная точность:**

   - Исключаются ошибки, связанные с пропуском препятствий.
   - Улучшается обработка взаимодействий объектов.

2. **Эффективность:**

   - Быстро движущиеся частицы обновляются чаще, тогда как медленные обрабатываются реже, что оптимизирует симуляцию.

3. **Физическая реалистичность:**

   - Напоминает замедление времени для объектов, движущихся с большими скоростями (аналогично эффектам теории относительности).

### Управление программой

- **Колёсико мыши** изменяет масштаб.
- **Клавиши AWSD** отвечают за перемещение камеры.
- **Стрелки вверх\вниз** меняют частоту рендера.
- **Стрелки вправо\влево** меняют перемещение частицы за тик.
